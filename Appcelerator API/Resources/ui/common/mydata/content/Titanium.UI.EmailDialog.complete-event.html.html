complete

Event of Titanium.UI.EmailDialog.
PlatformSince
Android0.8
iPhone0.8
iPad0.8
Mobile Web1.8

Summary

Fired when this email dialog has completed sending an email.


Description

On Mobile Web, as the platform cannot determine whether a message has been sent or not, this event fires immediately after calling open() and always reports success.


Properties

NameTypeSummary
errorString

Error message generated during an email send failure, or null if successful.

resultNumber

Status of the email send process. One of SENT, FAILED, SAVED (iOS only) or CANCELLED (iOS only).

Note that on Android this property will return SENT when the user discards or saves a message.

sourceObject

Source object that fired the event.

successBoolean

Indicates whether sending the email was successful.

typeString

Name of the event fired.
